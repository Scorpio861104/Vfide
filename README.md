## 🌟 What is VFIDE?

VFIDE is revolutionizing commerce by making **trust the currency of the future**. Built as a multi-chain protocol on Base, Polygon, and zkSync, VFIDE creates a world where honest behavior is rewarded, scammers are excluded, and everyone benefits from a more trustworthy marketplace.

> *"In a world of anonymous transactions, VFIDE proves that you can be trusted."*

### Key Features

- 🏦 **Personal Vaults** — Non-custodial smart contract wallets with social recovery
- 📊 **ProofScore™** — Dynamic reputation system (0-10,000) that rewards honest behavior
- 🛒 **Escrow System** — Protected commerce with dispute resolution
- 🗳️ **DAO Governance** — Community-driven decision making via elected councils
- 💰 **VFIDE Token** — Deflationary utility token with staking and governance rights
- 🛡️ **Guardian Network** — Trusted community members earn rewards by protecting vaults
- ⚡ **Real-Time Updates** — WebSocket-powered notifications and live data
- 🔗 **Multi-Chain** — Deployed on Base, Polygon, and zkSync

## 🎯 How It Works

### Your ProofScore™

Every VFIDE user has a **ProofScore** — a dynamic reputation score from 0 to 10,000 that reflects your on-chain behavior:

| Score Range | Trust Level | Benefits |
|-------------|-------------|----------|
| 800+ | **Verified** | Access to marketplace |
| 600+ | **Trusted** | Become a merchant |
| 500+ | **Guardian** | Earn by protecting others |
| 700+ | **Voter** | Participate in governance |

**Your score increases when you:**
- ✅ Complete successful transactions
- ✅ Hold and stake VFIDE tokens
- ✅ Participate in governance
- ✅ Help others as a Guardian or Mentor

**Your score decreases from:**
- ❌ Failed transactions or disputes
- ❌ Inactivity
- ❌ Community reports

### Development Setup

#### Prerequisites

- **Node.js 18+** (LTS recommended)
- **npm 9+** or **yarn 1.22+**
- **Git**
- **MetaMask** or another Web3 wallet
- **Foundry** (for contract development): `curl -L https://foundry.paradigm.xyz | bash`

#### Installation

```bash
# Clone the repository
git clone https://github.com/vfide/vfide.git
cd vfide

# Install frontend dependencies
cd frontend
npm install

# Copy environment variables
cp .env.example .env.local

# Edit .env.local with your configuration:
# - NEXT_PUBLIC_CHAIN_ID: 84532 (Base Sepolia testnet)
# - NEXT_PUBLIC_CONTRACT_ADDRESS: Your deployed contract address
# - NEXT_PUBLIC_WALLETCONNECT_PROJECT_ID: From WalletConnect Cloud
# - NEXT_PUBLIC_WEBSOCKET_URL: ws://localhost:3001 (for local WebSocket server)

# Start the development server
npm run dev
```

#### WebSocket Server (Optional)

The WebSocket server provides real-time updates for proposals, votes, chat, and notifications.

```bash

npm install

# Copy environment variables
cp .env.example .env
```

### Architecture Overview

VFIDE follows a modular architecture with clear separation of concerns:

1. **Frontend Layer** (Next.js)
   - Server-side rendering for performance
   - Client-side wallet integration via Wagmi
   - Real-time updates via WebSocket client
   - Responsive design with mobile-first approach

2. **WebSocket Layer** (Socket.IO)
   - Real-time event streaming
   - JWT + Ethereum signature authentication
   - Rate limiting and DDoS protection
   - Horizontal scaling support with Redis

3. **Smart Contract Layer** (Solidity)
   - Non-custodial personal vaults
   - ProofScore reputation algorithm
   - DAO governance with elected councils
   - Escrow and dispute resolution
   - Multi-chain deployment (Base, Polygon, zkSync)

4. **Blockchain Layer** (Ethereum L2s)
   - Base: Primary deployment
   - Polygon: High-throughput transactions
   - zkSync: Privacy-focused operations

See [Architecture Documentation](ARCHITECTURE.md) for detailed diagrams and explanations.
