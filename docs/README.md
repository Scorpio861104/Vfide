## 📝 Documentation Standards

When adding new documentation:

1. **Location**: Choose appropriate directory
   - Feature guides → `/docs/guides/`
   - Testing docs → `/docs/testing/`
   - Quick references → `/docs/reference/`
   - Temporary/status → `/docs/archive/`

2. **Format**: Use markdown with clear structure
   - Descriptive title (H1)
   - Table of contents for long docs
   - Code examples with syntax highlighting
   - Clear section headings

3. **Links**: Use relative links
   - Within docs: `[Guide](guides/GUIDE-NAME.md)`
   - To root: `[README](../README.md)`

4. **Updates**: Keep documentation current
   - Update when features change
   - Mark deprecated sections clearly
   - Archive outdated documents
